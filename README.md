#  ClubManagementSystem (Project 3)

Об'єктно-орієнтована консольна система на базі **C#** для комплексного обліку позашкільної освіти: менеджменту гуртків, координації розкладу та моніторингу відвідуваності.

Проєкт спроєктовано за принципом розділення обов'язків (Separation of Concerns):
* `Models` — доменні моделі сутностей та зв'язків.
* `DataStorage` — шар збереження даних (In-Memory колекції зі статичним доступом).
* `Services` / `UI` — бізнес-логіка застосунку та консольний інтерфейс користувача.


## 🦾 Хто рулить системою?

Програма підлаштовує свій функціонал під три ключові ролі:

###  Роль: Учень / Батько
* **Гнучкий пошук:** Індексація та пошук гуртків за текстовим збігом (назва / адреса) без урахування регістру літер.
* **Менеджмент записів:** Реєстрація на секції за унікальним ID із пре-валідацією на дублікати зв'язків.
* **Таймлайн:** Генерація індивідуального розкладу занять, автоматично відсортованого за вектором часу.
* **Трекінг відвідуваності:** Перегляд логів присутності (*Present / Absent*) з калькуляцією фінального KPI відвідуваності.

###  Роль: Викладач
* **Пакетне логування:** Швидка фіксація відвідуваності шляхом введення ID присутніх студентів через пробіл (іншим автоматично присвоюється статус *Absent*).
* **Списки груп:** Доступ до списків студентів, закріплених за гуртком викладача, разом із контактними даними.
* **Експрес-аналітика:** Генерація звітів щодо заповнюваності груп та активності на конкретних сесіях.

###  Роль: Адміністратор
* **CRUD Інфраструктури:** Повний цикл керування картками викладачів та реєстрація нових гуртків/напрямків.
* **Оркестрація сесій:** Планування конкретних занять у часовому просторі (дата, таймінги) з прив'язкою до аудиторного фонду.
* **Конфлікт-менеджмент:** Оперативне перевизначення кабінетів для активних сесій у разі виникнення накладок розкладу.
